## 글로벌 레이아웃 구조

```
┌────────────────────────────────────────────────────────────────────┐
│ HEADER: 서비스명 | 네비 탭(메인/매장관리/로그) | WS 연결상태 | 🔔 소리토글 | 매장패널토글 | [로그아웃] │
├────────────────────────────────────────────────────────────────────┤
│                         CONTENT AREA                               │
│ (각 페이지별 내용)                                                   │
└────────────────────────────────────────────────────────────────────┘
```

**헤더 우상단 버튼 (구현됨):**
- **WS 연결 배지**: `✓ 연결됨` / `연결 중...` / `연결 끊김` 상태 표시
- **🔔 알림 소리 토글**: ON/OFF, `localStorage` 저장
- **매장 패널 토글**: 메인 화면의 우측 매장 그리드 숨기기/보이기
- **로그아웃** 버튼

**네비게이션:** 헤더 상단 탭 방식 (`/`, `/stores`, `/logs`)
- 현재 활성 탭: 하단 보더 강조
- 로그인 페이지에서는 네비 숨김

## 메인 관제 화면 레이아웃 (`/`)

```
┌─────────────────────────────────────────────────────────────────┐
│ [헤더]                                                           │
├──────────────────────┬───┬──────────────────────────────────────┤
│  알림 리스트 (기본 60%) │ ↔ │  매장 상태 그리드 (기본 40%)          │
│  PENDING ALERTS (N)  │드 │  STORES N/M                         │
│                      │래 │                                      │
│  [붉은행] 강남점       │그 │  [카드] 강남점    ● ONLINE  ★★★      │
│  냉장고1 센서 off→on  │   │  [카드] 홍대점    ○ OFFLINE ★★★      │
│  22:05:01            │   │                                      │
│  ...                 │   │                                      │
└──────────────────────┴───┴──────────────────────────────────────┘
```

**드래그 분할**: 중앙 구분선을 드래그하여 좌우 비율 조절 가능. 비율은 `localStorage` 저장.  
**매장 패널 숨김 시**: 알림 리스트가 전체 폭 차지.

### 알림 리스트
- 미확인 alert를 최신순으로 표시
- 각 row: `[매장명] · [센서명] | [타입] · [상태변화] | [발생시각]`
- 중요도 4~5 row: **붉은 배경** (`--alert-row-high`) + 좌측 4px `--accent` 보더
- 클릭 시 → 영상 팝업 오픈 (해당 알림 인덱스로)
- 알림 없음: "현재 미확인 알림이 없습니다" 표시

### 매장 상태 그리드
- 카드 구성: `[ONLINE/OFFLINE 배지] 매장명 | 중요도(★N) | entity N개`
- **정상 온라인 카드**: opacity/color 낮춰 배경으로 물러남 (Dead-state)
- **오프라인 카드**: 흐리게 처리 + 배지 표시
- 레이아웃: 2열 그리드 (현재 구현)

### 실시간 WS 이벤트 처리
| 이벤트 | 처리 |
|---|---|
| `init` | 전체 상태 초기화 (stores + pending_alerts) |
| `alert.new` | 리스트 최상단 추가. `is_in_schedule=true` + 소리토글 ON → 소리 재생. `importance=5` → 팝업 자동 오픈 |
| `alert.acknowledged` | 해당 alert_id 행 즉시 제거 |
| `entity.update` | 해당 매장 카드 센서 상태 갱신 |
| `store.status` | 온라인/오프라인 배지 갱신 |

### 영상 팝업 레이아웃 (구현됨)

```
┌────────────────────────────────────────────────────────┐
│  강남점 · 냉장고1 센서   off→on   22:05:01      [✕]    │
├────────────────────────────────────────────────────────┤
│  CH1 CH1 | CH2 CH2 | CH3 CH3 | ... (2개 이상 카메라 시) │
├────────────────────────────────────────────────────────┤
│                                                        │
│     [go2rtc stream.html iframe / 16:9 비율]            │
│     (alert.stream_url 있을 때 자동 선택 채널 표시)      │
│                                                        │
├────────────────────────────────────────────────────────┤
│  ← 이전   ↑↓ 이전/다음 · Enter 확인 · Esc 닫기  다음 → │
│                                    [✓ 확인]            │
└────────────────────────────────────────────────────────┘
```

- 팝업 크기: 최대 800px, 세로 중앙 정렬, 반투명 오버레이
- **영상**: go2rtc `stream.html?src={stream_source}` iframe 삽입
  - `alert.stream_url` 있을 때 해당 채널 자동 선택
  - `stream_url` null이면 "등록된 카메라가 없습니다" 표시
  - 매장에 카메라 2개 이상 → 채널 탭 버튼으로 전환 가능
- **[✓ 확인]** 클릭 → `POST /alerts/{id}/acknowledge` → WS 브로드캐스트 → 팝업 닫힘
- 키보드: ↑↓ = 이전/다음 알림, Enter = 확인, Esc = 닫기
- `role="dialog"`, `aria-modal="true"`, 열릴 때 포커스 이동

## 매장 리스트 (`/stores`)

- 전체 매장 테이블: 매장명, 주소, 온라인 상태, 중요도
- 행 클릭 → `/stores/{store_id}` 이동
- `+ 매장 추가` 버튼

## 매장 상세 설정 (`/stores/{store_id}`)

상단: `매장명 [ONLINE/OFFLINE 배지] #store_id` + 4탭

### 기본 정보 탭 (구현됨)
- 매장명, 주소, 기기 SN 수정
- **GO2RTC URL** 입력 (예: `http://192.168.0.x:1984`) — 변경 시 모든 카메라 stream_url 자동 재계산
- 중요도 선택 (1~5 별 클릭)
- **강제 알림 설정** 3-way toggle: `스케줄 따름` / `강제 ON` / `강제 OFF`
- 저장 버튼

### CCTV 설정 탭 (구현됨)
- `go2rtc_url` 미설정 시 안내 메시지
- 카메라 채널 목록: `CHN | 채널명 | stream_source`
- 각 채널: `▶ 보기` (iframe 토글) / `수정` / `삭제`
- **채널 추가 폼**: 채널 번호, 이름, **스트림 소스** (go2rtc 소스명, 예: `ch1_sub`)
  - ~~RTSP URL 입력~~ → go2rtc가 RTSP를 처리하므로 Frontend는 소스명만 저장
- `+ 채널 추가` 버튼

> **stream_url** = `go2rtc_url` + `/stream.html?src=` + `stream_source` (Backend가 조합)

### 알림 시간 탭 (구현됨)
- `↻ 외부 서버에서 동기화` 버튼
- 요일별 (월~일) 시작/종료 시간 + 활성화 토글 + 저장 버튼
- **시프트 기준 안내 문구** (하단 표시):
  - 각 요일 스케줄은 전날 22:00 ~ 당일 12:00
  - 예) "월 스케줄 = 일요일 22:00 ~ 월요일 12:00"

> **시프트 기준 (Backend 결정)**: 22:00 이후는 다음날 시프트 처리  
> 일요일 23:00 → 월요일 시프트. 월요일 22:00 → 화요일 시프트.

### 센서·릴레이 탭 (구현됨)
- **등록된 ENTITY** 목록 (센서/릴레이 구분)
  - 각 행: `[타입배지] 커스텀명 | ha_entity_id | 현재상태 | 연결카메라CH`
  - **수정 폼**: 이름, 타입 선택, 알림 발생, **카메라 채널 연결** 드롭다운 (카메라 목록 기반)
  - **해제** 버튼 (confirm 다이얼로그 후 삭제)
  - **릴레이**: ON/OFF 토글 버튼 → `POST /relays/{ha_entity_id}/command`
- **HA에서 Entity 가져오기** 버튼
  - 미등록 HA entity 목록 하단 표시
  - 각 미등록 entity에 **등록** 버튼 → 폼 펼침: 이름, 타입, 알림 발생 → 등록 확인
  - ※ 신규 등록 폼에는 camera_channel 없음 — 등록 후 수정 폼에서 연결
- **ENTITY 타입 관리** (탭 하단): 커스텀 타입 추가/삭제

## 로그 화면 (`/logs`)

**목적**: 이벤트 이력 조회

### 필터 (구현됨)
- 매장 선택 (드롭다운)
- 타입명 검색 (텍스트)
- 상태 from / to 검색 (텍스트)
- 기간 from ~ to (datetime-local picker)
- 조회 / 초기화 버튼

### 테이블 컬럼
`발생시각 | 매장명 | 센서명 | 타입 | 변화 (off→on)`

- 페이지네이션 (현재 구현됨)
- CSV 다운로드: 미구현

## 디자인 토큰

```css
--bg-base: #0f1117
--bg-surface: #1a1d27
--accent: #e63946
--success: #2ec4b6
--warning: #f4a261
--text-primary: #f1f3f5
--text-muted: #868e96
--alert-row-high: #3d0e0e   /* 중요도 4~5 배경 */
```

**폰트:**
- Pretendard: 한국어 UI 본문
- JetBrains Mono: 타임스탬프, 데이터 수치
- Syne: 수치/헤더 (DESIGN.md 기준)

**핵심 디자인 원칙:**
1. 색상은 상태를 나타낼 때만 (장식적 색상 금지)
2. Dead-state 카드: 정상 온라인 매장 카드는 opacity/color 낮춤
3. 이중 강조: 중요도 4~5 알림 행 → `--alert-row-high` 배경 + 좌측 4px `--accent` 보더

## 반응형 브레이크포인트

| 브레이크포인트 | 메인 화면 | 매장 상세 |
|---|---|---|
| ≥1440px | 알림60% + 그리드40% (드래그 조절 가능) | 탭 + 콘텐츠 전체폭 |
| 1024~1439px | 알림55% + 그리드45% | 탭 + 콘텐츠 전체폭 |
| <1024px | 알림 전체폭 → 매장 그리드 하단 스크롤 | 탭 수평 스크롤 |
| <768px | 단일 컬럼 | 탭 드롭다운 or 아코디언 |

> **미구현**: 반응형 레이아웃은 현재 미완성 (후반 작업 예정)

## Interaction State 정의

| 기능 | LOADING | EMPTY | ERROR |
|---|---|---|---|
| 초기 로딩 (`/`) | WS 연결 중 배지 | "현재 미확인 알림이 없습니다" | WS 끊김: 헤더 배지 |
| HA Entity 조회 | "HA에서 불러오는 중..." | "미등록 entity가 없습니다" | "HA 응답 없음" |
| 알림 확인 버튼 | disabled + 스피너 | — | — |
| 릴레이 제어 | disabled + 스피너 | — | — |
| 로그 조회 | 테이블 로딩 | "조건에 맞는 이벤트가 없습니다" | — |

## 알림 소리 조건

- `is_in_schedule=true` + 소리 토글 ON → 소리 재생
- 화면 표시(리스트 추가)는 `is_in_schedule` 무관 항상
- `importance=5` 알림 → 팝업 자동 오픈 (미확인 팝업 없을 때)
