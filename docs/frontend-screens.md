# Frontend 화면 설계

## 화면 목록

```
/login                    로그인
/                         메인 관제 화면
/stores                   매장 리스트
/stores/{store_id}        매장 상세 설정
/logs                     이벤트 로그
```

## 글로벌 레이아웃 구조

```
┌──────────────────────────────────────────────────────────────────┐
│ HEADER: 서비스명 | 네비 탭(메인/매장관리/로그) | 연결상태 | 사용자명 [로그아웃] │
├──────────────────────────────────────────────────────────────────┤
│                       CONTENT AREA                               │
│  (각 페이지별 내용)                                                │
└──────────────────────────────────────────────────────────────────┘
```

**네비게이션:** 헤더 상단 탭 방식 (`/`, `/stores`, `/logs`)
- 현재 활성 탭: 하단 보더 강조 or 배경색 구분
- 로그인 페이지에서는 네비 숨김

## 디자인 토큰 (최소 정의)

```
배경색:   --bg-base: #0f1117 (다크)  또는  #f8f9fa (라이트, 선택사항)
표면색:   --bg-surface: #1a1d27
강조색:   --accent: #e63946  (알림/위험)
성공색:   --success: #2ec4b6 (온라인)
경고색:   --warning: #f4a261 (중간 중요도)
텍스트:   --text-primary: #f1f3f5  --text-muted: #868e96
알림 행:  --alert-row-high: #3d0e0e  (중요도 4~5, 붉은 배경)
타임스탬프 폰트: 모노스페이스 (JetBrains Mono, Fira Code, 또는 monospace 스택)
```

> 다크/라이트 모드 결정: 관제실 환경(저조도)을 고려하면 다크 모드 기본 권장.
> 단순화 위해 단일 테마(다크)로 시작, 추후 테마 전환 추가 가능.

## Interaction State 정의

| 화면/기능 | LOADING | EMPTY | ERROR | SUCCESS |
|---|---|---|---|---|
| 초기 로딩 (`/`) | 스켈레톤 UI (알림 리스트 + 매장 그리드 자리) | 알림 없음: "현재 미확인 알림이 없습니다" + 매장 그리드 표시 유지 | WS 연결 실패: 헤더에 "⚠ 서버 연결 끊김" 배너 + 재연결 버튼 | — |
| WS 재연결 중 | 헤더 배너: "🔄 재연결 중..." (자동 재시도) | — | 30초 이상 실패 시 "수동 새로고침" 버튼 노출 | "✓ 연결됨" 1초 표시 후 배너 사라짐 |
| HA Entity 조회 | "HA에서 불러오는 중..." 스피너 (최대 10초) | "등록된 entity가 없습니다. HA에 센서를 추가하세요." | 10초 timeout: "HA 응답 없음. Edge 온라인 상태를 확인하세요." | entity 목록 표시 |
| 알림 확인 버튼 | 버튼 disabled + 스피너 | — | "확인 처리 실패. 다시 시도하세요." | 행 즉시 제거 (WS 브로드캐스트로) |
| 릴레이 제어 | 버튼 disabled + 스피너 | — | "명령 전송 실패" 토스트 | 상태 즉시 갱신 (entity.update WS) |
| 매장 목록 (`/stores`) | 스켈레톤 테이블 행 5개 | "등록된 매장이 없습니다. 첫 매장을 추가하세요." + 추가 버튼 | API 오류: "매장 목록을 불러오지 못했습니다. 새로고침" | — |
| 로그 조회 (`/logs`) | 테이블 스켈레톤 | "조건에 맞는 이벤트가 없습니다." | API 오류 토스트 | 결과 테이블 표시 |
| 카메라 미리보기 | 버퍼링 스피너 | — | "스트림을 불러올 수 없습니다. go2rtc URL 확인" | 영상 재생 |

## 1. 로그인 (`/login`)

- username / password 입력
- 로그인 성공 → `/` 리다이렉트
- 세션 만료 시 자동 리다이렉트

---

## 2. 메인 관제 화면 (`/`)

**목적**: 알림이 활성화된 매장의 실시간 이벤트를 즉시 파악

### 알림 리스트
- 미확인 alert_events를 최신순으로 표시
- 각 row: `[매장명] [센서명] [상태변화] [발생시각]`
- 중요도 4~5인 매장의 알림 row: **붉은 배경**
- 클릭 시 → 해당 매장 go2rtc 영상 팝업 + 확인 버튼
- 확인 버튼 클릭 → WS를 통해 전체 접속자에게 즉시 반영 (영상 닫힘)

### 매장 상태 요약
- 전체 매장 온라인/오프라인 현황 (카드 or 그리드)
- 각 카드: 매장명, 상태, 최근 이벤트, 중요도 배지

### 실시간 동작
- WebSocket 연결 유지
- `alert.new` 수신 → 알림 리스트 최상단에 추가 (관제시간 내인 경우에만 소리 알림)
- `alert.acknowledged` 수신 → 해당 row 즉시 확인 처리
- `entity.update` 수신 → 매장 카드 상태 갱신
- `store.status` 수신 → 온라인/오프라인 배지 갱신
- `store.status = offline` 수신 → 해당 매장 카드에 눈에 띄는 오프라인 표시 + 헤더 상단 "⚠ {매장명} 오프라인" 토스트 알림

### 영상 팝업 레이아웃

알림 행 클릭 시 표시:

```
┌───────────────────────────────────────────────────────┐
│  강남점 — 출입구 도어 열림          [X 닫기]           │
│  22:05:01                                             │
├───────────────────────────────────────────────────────┤
│                                                       │
│           [go2rtc WebRTC 영상 플레이어]                │
│           (16:9 비율, 전체 팝업 폭)                    │
│                                                       │
├───────────────────────────────────────────────────────┤
│  이전 알림 <<              >> 다음 알림   [✓ 확인]    │
└───────────────────────────────────────────────────────┘
```

- 팝업 크기: 화면 폭 최대 800px, 세로 중앙 정렬
- 배경: 반투명 오버레이 (dim)
- 영상 로딩 실패 시: "스트림 연결 실패" 메시지 + 재시도 버튼
- [✓ 확인] 클릭 → 팝업 닫힘 + WS acknowledge 전송
- 다중 알림 시 이전/다음 알림 탐색 가능

---

## 3. 매장 리스트 (`/stores`)

- 전체 매장 테이블: 매장명, 주소, 온라인 상태, 중요도, 등록일
- 매장 클릭 → `/stores/{store_id}` 이동
- 매장 추가 버튼

---

## 4. 매장 상세 설정 (`/stores/{store_id}`)

### 기본 정보 탭
- 매장명, 주소, 기기 SN 수정
- 중요도 선택 (1~5 별점 or 드롭다운)
- 저장 버튼

### CCTV 설정 탭
- go2rtc URL (Edge IP) 설정
- 카메라 채널 목록 (channel 번호, 이름, RTSP main/sub URL)
- 채널 추가/수정/삭제 → 저장 즉시 Backend가 go2rtc API 호출하여 반영
- 각 채널 미리보기 버튼 (go2rtc WebRTC 스트림)

### 알림 시간 설정 탭
- 요일별 (월~일) 시작/종료 시간 입력
- 외부 관제서버 연동 표시: 마지막 폴링 시각, 현재 적용된 값
- "지금 동기화" 버튼 → `/api/v1/stores/{store_id}/schedules/sync`
- **강제 알림 설정**: 없음(스케줄) / 강제 ON / 강제 OFF (3-way toggle)

### 센서·릴레이 설정 탭
- "HA에서 Entity 가져오기" 버튼 → `/api/v1/stores/{store_id}/ha/entities` 조회
- 조회된 전체 entity 리스트 표시 (ha_entity_id, 현재 상태)
- 각 entity에 대해:
  - 등록 여부 toggle
  - 사용자 명칭 입력 (custom_name)
  - 타입 선택 (entity_types 목록, 사용자 추가 가능)
  - 알림 트리거 여부 toggle (triggers_alert)
- 저장 버튼

### 타입 관리 (센서·릴레이 탭 내 또는 별도)
- 사용자 정의 타입 추가/삭제 (기본 타입: 출입문/냉장고/카운터/기타는 삭제 불가)

---

## 5. 로그 화면 (`/logs`)

**목적**: 3개월치 상태 변화 이력 조회

### 필터
- 매장명 검색 (텍스트)
- 타입 선택 (entity_types 목록)
- 상태 변화 방향: 전체 / 열림→닫힘 / 닫힘→열림
- 기간 (from ~ to datepicker)

### 테이블 컬럼
`발생시각 | 매장명 | 센서명 | 타입 | 변화 (닫힘→열림)` 

- 페이지네이션
- CSV 다운로드 (선택사항 — TBD)

## 알림 소리 조건

관제시간 판단은 Frontend가 아닌 **Backend**가 판단 후 WS로 전송.
- `alert.new` 이벤트에 `is_in_schedule: true/false` 포함
- Frontend는 `is_in_schedule=true` 인 경우에만 소리 재생
- 화면 표시(리스트 추가)는 is_in_schedule 무관하게 항상

### 알림 소리 ON/OFF 토글

- 위치: 헤더 우상단 🔔 아이콘 버튼 (ON 시 활성, OFF 시 취소선 or 슬래시)
- **기본값: ON**
- ON 상태 + `is_in_schedule=true` 인 `alert.new` 수신 시 소리 재생
- 브라우저 autoplay 정책: 사용자가 토글 버튼을 한 번 클릭하면 자동으로 해결됨
- 새로고침 시 상태 유지: `localStorage`에 저장

## 매장 상태 그리드 명세

- **레이아웃:** 4열 그리드, 세로 스크롤 (100개 매장도 동일 구조)
- **정렬:** 중요도 내림차순 → 오프라인 하단
- **카드 구성:** `[온라인/오프라인 배지] 매장명 | 중요도(★N) | 마지막 이벤트 시각`
- **카드 크기:** 고정 높이 80px 내외, 내용 잘림 처리 (tooltip으로 전체 표시)
- **오프라인 카드:** 흐리게 처리 (opacity 0.5) + 오프라인 배지
