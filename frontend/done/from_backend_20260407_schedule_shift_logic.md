# [Backend → Frontend] 관제 스케줄 시프트 기준 수정

**발신**: Backend  
**수신**: Frontend  
**일시**: 2026-04-07

---

## 변경 내용

관제 스케줄 판단 로직을 사용자 기준에 맞게 수정했습니다.

### 시프트 기준

| 스케줄명 | 관제 시간 |
|---------|----------|
| 월 | 일요일 22:00 ~ 월요일 12:00 |
| 화 | 월요일 22:00 ~ 화요일 12:00 |
| 수 | 화요일 22:00 ~ 수요일 12:00 |
| 목 | 수요일 22:00 ~ 목요일 12:00 |
| 금 | 목요일 22:00 ~ 금요일 12:00 |
| 토 | 금요일 22:00 ~ 토요일 12:00 |
| 일 | 토요일 22:00 ~ 일요일 12:00 |

**22:00 이후는 다음날 시프트**로 처리됩니다.

### 수정 전 (버그)
- 일요일 23:00 → "일" 스케줄 조회 ❌
- 월요일 22:00 → "월" 스케줄 조회 ❌

### 수정 후
- 일요일 23:00 → "월" 시프트 조회 ✅
- 월요일 22:00 → "화" 시프트 조회 ✅

---

## 프론트엔드 영향

`alert.new` 및 `init.pending_alerts`의 `is_in_schedule` 값이 이 기준으로 계산됩니다.

스케줄 설정 UI에서 요일 레이블 표시 시 위 시프트 기준으로 안내해 주세요.  
예) "월 스케줄 = 일요일 밤 ~ 월요일 낮 관제 시간"
